Learn Git - A Step-by-Step Guide

This guide covers the basics of Git, from installation to making changes. Follow these steps to improve your version control skills.

  1. Install Git: Download and install Git for your operating system from the official website: Git SCM Downloads.
  2. Basic Configuration: Set up your username and email in Git by running these commands in the terminal: 
    git config --global user.name "Your Name"
    git config --global user.name "Zahid"
    Add your email address:
    git config --global user.email "your.email@example.com"
    git config --global user.email "zahid@zahid.com"
    This links your commits to your identity.
  3. Initialize a Git Repository: Start a Git repository in your project folder: 
    git init
  4. Check Repository Status: View the status of your repository and track changes: 
    git status
  5. Add a Remote Repository: Link your local repository to a remote repository (e.g., GitHub): 
    git remote add origin repository-url
    git remote add origin https://github.com/username/repository.git
  6. Stage Files for Commit: Add files to the staging area for committing. You can add specific files or all changed files: 
    git add filename
    git add 'learn_git.html'
    git add .
  7. Commit Your Changes: Commit your staged changes with a message: 
    git commit -m "Describe your changes"
    git commit -m "Add initial guide content"
  8. Amend the Last Commit: Modify the most recent commit (e.g., change the commit message): 
    git commit --amend
    git commit --amend -m "Updated commit message"
  9. View Commit History: View a list of previous commits: 
    git log
    For a simplified view: 
    git log --oneline
  10. Create a New Branch: Create a new branch for new features or fixes: 
    git branch new-feature
    git branch feature-xyz
  11. Switch Between Branches: Move between branches: 
    git checkout branch-name
    git checkout main
    Alternatively: 
    git switch branch-name
    git switch main
  12. Merge Branches: Merge changes from a branch back into the main branch: 
    git merge branch-name
    git merge feature-xyz
  13. Push Changes to a Remote Repository: Push your changes to a remote repository (e.g., GitHub or GitLab): 
    git push origin branch-name
    git push origin main
    For the first push or to set up tracking for a new branch: 
    git push -u origin branch-name
    git push -u origin main
  14. Pull Changes from Remote Repository: Update your local repository with changes from the remote repository: 
    git pull
  15. Undo Local Changes: Discard changes in the working directory: 
    git checkout -- filename
    git checkout -- learn_git.html
  16. Unstage Changes: Unstage a file without losing its changes: 
    git reset filename
    git reset learn_git.html
  17. Remove Files from Git: Stop tracking a file but keep it in your local directory: 
    git rm --cached filename
    git rm --cached learn_git.html
  18. Rebase Branches: Reapply changes on top of another branch: 
    git rebase branch-name
    git rebase main
  19. Stash Changes: Temporarily save your work and switch branches: 
    git stash
    Retrieve stashed changes with: 
    git stash pop
  20. View Differences: See changes between commits: 
    git diff
  21. Keep Learning: Practice Git regularly and explore advanced features. Visit the Git documentation for more resources.
