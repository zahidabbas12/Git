Learn Git - A Step-by-Step Guide

This guide will teach you the basics of Git, from installation to committing changes and more.

  1. Install Git: Download and install Git for your operating system. Visit the official Git website: Git SCM Downloads.
  2. Basic Configuration: After installation, open a terminal and set up your username and email address by running these commands: 
    
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
    These settings ensure your commits are linked to your identity.
  3. Initialize a Git Repository: Navigate to your project folder in the terminal and initialize a Git repository by running: 
    git init
    This command creates a hidden `.git` folder where Git will store all your project’s version history.
  4. Check Repository Status: To view the current state of your repository and track changes, use: 
    git status
  5. Stage Files for Commit: Add files to the staging area to prepare them for committing. You can add specific files or all changed files: 
    git add filename
git add .
  6. Commit Your Changes: After staging your files, commit them with a meaningful message: 
    git commit -m "Describe your changes"
    This saves a snapshot of your changes.
  7. View Commit History: To see a list of previous commits, run: 
    git log
  8. Create a New Branch: To work on a new feature or fix a bug without affecting the main codebase, create a new branch: 
    git branch new-feature
  9. Switch Between Branches: To move between branches, use: 
    git checkout branch-name
  10. Merge Branches: After completing your changes in a branch, merge them back into the main branch: 
    git merge branch-name
  11. Push Changes to a Remote Repository: After committing changes locally, push them to a remote repository (e.g., GitHub or GitLab): 
    git push origin branch-name
    For the first time, or to set up tracking for a new branch, use: 
    git push -u origin branch-name
  12. Pull Changes from Remote Repository: To update your local repository with the latest changes from a remote one, use: 
    git pull
  13. Undo Local Changes: If you want to discard changes you’ve made in the working directory, run: 
    git checkout -- filename
  14. Unstage Changes: To unstage a file without losing its changes, use: 
    git reset filename
  15. Remove Files from Git: If you want to stop tracking a file but keep it in your local directory, use: 
    git rm --cached filename
  16. Keep Learning: The best way to get comfortable with Git is to keep practicing. Start with small projects and explore advanced features like `git diff`, `git fetch`, and more. For additional resources, visit the Git documentation.
